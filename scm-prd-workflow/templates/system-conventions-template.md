## 2. 通用组件

### 2.1 搜索组件

- 标准筛选项类型：{如：文本输入、下拉单选、下拉多选、日期范围、数值范围}
- 筛选项布局：{如：一行展示N个，超出折叠}

### 2.3 审批组件

## 3. 交互规范

### 3.1 确认弹窗

- 触发场景：{如：删除、批量操作、不可逆操作时弹出二次确认}
- 弹窗格式：{如：标题 + 影响说明 + 确认/取消按钮}

### 3.2 反馈提示

- 成功提示：{如：右上角绿色Toast，3秒自动消失}
- 错误提示：{如：右上角红色Toast，需手动关闭}
- 加载状态：{如：按钮Loading态 + 页面骨架屏}

### 3.3 批量操作

- 标准模式：{如：勾选行 → 底部浮动操作栏出现 → 选择操作 → 二次确认}
- 结果反馈：{如：操作完成后展示成功/失败数量汇总}

## 4. 权限模型

### 4.1 标准RBAC模式

- 角色体系：{如：超级管理员 / 部门管理员 / 普通用户}
- 功能权限：{如：菜单可见性 + 按钮操作权限}
- 数据权限：{如：按组织架构层级控制数据可见范围}

### 4.2 默认权限规则

- 只读默认：{如：所有角色默认有查看权限，编辑/删除需显式授权}
- 审批权限：{如：审批操作仅对审批角色可见}
- 数据导出：{如：导出权限独立控制}

## 5. 接口公约

### 5.1 错误处理

- 标准错误码：{如：400参数错误 / 401未认证 / 403无权限 / 404不存在 / 500服务端错误}
- 错误响应格式：{如：`{ code, message, details }`}
- 重试策略：{如：5xx错误自动重试3次，间隔指数退避}

### 5.2 分页

- 标准分页参数：{如：`page` + `pageSize`，默认pageSize=20}
- 响应格式：{如：`{ list, total, page, pageSize }`}

### 5.3 认证

- 认证方式：{如：JWT Token，Header: Authorization: Bearer {token}}
- Token刷新：{如：过期前自动刷新 / 过期后跳转登录}

## 6. 数据公约

### 6.1 命名规范

- 字段命名：{如：snake_case，英文全拼，禁止缩写}
- 枚举值命名：{如：UPPER_SNAKE_CASE}

### 6.2 审计字段

- 标准审计列：{如：created_by, created_at, updated_by, updated_at}
- 记录方式：{如：框架自动填充，不需要业务代码处理}

### 6.3 软删除

- 标准模式：{如：`is_deleted` 字段标记，查询自动过滤}
- 物理删除：{如：仅DBA手动执行，业务层不提供物理删除}

## 7. 非功能默认值

### 7.1 性能

| 场景 | 默认SLA |
|------|--------|
| 页面加载 | {如：≤2秒} |
| 列表查询 | {如：≤1秒（1000条以内）} |
| 表单提交 | {如：≤3秒} |
| 批量操作 | {如：≤10秒（100条以内）} |
| 报表生成 | {如：≤30秒} |
| 导出 | {如：≤60秒（10000条以内）} |

### 7.2 可用性

- 系统可用性：{如：99.9%}
- 计划维护窗口：{如：每周三凌晨2:00-4:00}

### 7.3 安全

- 数据脱敏：{如：手机号中间4位 / 身份证中间8位 / 银行卡保留前6后4}
- 操作日志：{如：所有增删改操作自动记录，含操作人、时间、变更前后值}
- 敏感操作：{如：批量删除、数据导出、权限变更需二次认证}
