# PRD 撰写 — 叙事规划核心原则

读取时机：Stage B 叙事规划阶段。Stage C 进入后可释放。

## 目标

基于Stage A的完整需求信息，产出**重点清晰、逻辑连贯、可直接用于开发评审**的PRD文档。PRD是讲故事，不是填模板——先想清楚"这份文档要怎么讲"，再动笔。

---

## 叙事规划（撰写前的产品经理思考）

### 叙事规划的产出

叙事规划模板（核心叙事 + 方案评估 + 已知背景 + 维度详略 + 图表/原型策略 + 章节映射 + ID策略）的完整格式见下方。NP 自检和 CD-01 定义见 `writing-narrative.md`。

### 核心叙事（一段话）

{什么问题} → {怎么解决} → {为什么这样做} → {预期价值}

### 方案评估

在确定核心叙事后、展开文档结构前，退后一步审视方案本身。

**因果链自检**：从 intake 痛点到当前方案，因果链是否每一步都成立？
- {痛点} → 导致 → {问题} → 所以需要 → {方案} → 预期达到 → {效果}
- 如果任何一步逻辑跳跃，**修改方案或完善表达使逻辑完整**——不是打补丁加解释段落

**方案合理性**：最合理的方案？更简单的替代方案？选择逻辑？复杂度与问题匹配？

**成本与权衡**：牺牲什么？值得吗？intake 中用户的担忧/约束是否已回应？

**执行方式**：评估结果**融入核心叙事和维度选择**。AI 能自行调整的直接修改；需要用户输入的暂停呈现问题。

### 已知背景推断

**目的**：识别"所有目标读者都已经知道的"，PRD 中引用但不展开——减少补偿性冗余。

**注意**：PRD 的读者不只是撰写者，还包括开发、QA、架构师、BA、决策者。"已知"是指**所有目标读者都知道**，不是作者知道就省略。

**执行方式**：AI 主动推断已知背景假设让用户确认（不向用户提问"你知道什么"）。

**各模式差异**：自主模式写入 intake.md `known_context` 字段；轻量模式不显式处理。

### 表达维度与注意力预算

表达维度由**叙事主次**（故事中的角色）× **读者认知**（了解程度）交叉决定：

| | 读者不了解 | 部分读者了解 | 团队/行业共识 |
|---|---|---|---|
| **叙事核心** | ★★★ 重点展开 | ★★ 标准描述 | ★ 简述 |
| **支撑上下文** | ★★ 标准描述 | ★ 简述 | 引用不解释 |
| **非核心** | ★ 简述 | 引用或省略 | 省略 |

### 重点、模块间逻辑、图表策略、原型策略

- **重点**：读者必须理解的 2-3 件事
- **模块间逻辑主线**：读者应按什么顺序理解
- **图表策略**：复用 `diagram-patterns.md` 成本收益判断
- **原型策略**：触发条件详见 `prototype-planning.md`
- **快速线框**（可选，命中≥3条触发条件时）：1-2个关键屏幕的文字线框描述

### 章节编号映射 + ID 使用策略

根据维度裁剪结果确定最终章节编号（连续不跳号）。ID 使用策略：第一层（始终）G-XX/F-XXX；第二层（按需）C-XX/IF-XXX。
